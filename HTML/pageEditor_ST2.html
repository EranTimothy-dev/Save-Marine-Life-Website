Home Page

Technical Description

The Home page consists of three main sections: the navbar, the main content section, and the footer. The navbar ensures easy navigation across different pages and follows a global CSS styling with JavaScript for responsiveness on mobile devices.

The main content section is divided into multiple subsections:

  • Hero Section: Displays a visually appealing background image, a prominent heading, and a call-to-action scroll-down button.
  • Mission Section: Contains an image and a text description highlighting the purpose of the initiative.
  • Help Section: Provides ways users can contribute, using icons and brief descriptions.
  • Explore Section: Offers further reading with links to detailed content pages.

The styles for the main content section are defined in a separate CSS file, homepage.css, which includes styling for the hero section, mission container, help section, and explore section.

Accessibility

Accessibility considerations include:

  • Using semantic HTML elements (e.g., <nav>, <main>, <section>) for better screen reader support.
  • Ensuring text contrast with background colors for readability.
  • Providing alt attributes for all images to aid visually impaired users.

Table Page

Technical Description

The Table Page presents marine conservation efforts in a structured tabular format, enhancing readability and accessibility. The page follows a clean and responsive design using HTML and CSS.

Key HTML elements used:

  • <table> - Organizes data into rows and columns for clear presentation.
  • <thead> & <tbody> - Separates table headers from content, improving structure.
  • <th> - Defines header cells with bold text for better clarity.
  • <td> - Holds data within the table, allowing row and column-based styling.
  • <a> - Provides external links for additional resources.

Key CSS properties used:

  • border-collapse: collapse; - Removes extra spacing between table borders for a cleaner look.
  • box-shadow - Adds depth to the table, making it visually distinct.
  • nth-child selectors - Alternates row colors to improve readability.
  • hover effects - Enhances user experience by highlighting rows when hovered over.
  • media queries - Ensures the table adapts well to different screen sizes.

Accessibility

Accessibility features include:

  • Use of th elements for headers to provide context for screen readers.
  • Contrast between text and background for better readability.
  • Responsive design ensures the table remains navigable on mobile devices.
  • Clickable links for external resources are clearly labeled.

Content Page

Technical Description

The Content Page is structured using HTML and styled with a combination of a dedicated CSS file and a global stylesheet for consistency across the website. The page contains a hero section, content sections, and an interactive explore section.

Key HTML elements used:

  • <section> - Organizes content into meaningful sections for better readability and accessibility.
  • <div> - Used to create containers for different sections like mission, help, and explore.
  • <h1> & <h2> - Defines hierarchical headings to structure content properly.
  • <a> - Provides navigation links to other pages.
  • <img> - Displays visual content such as mission-related images and icons.

Key CSS properties used:

  • background-image - Sets a gradient overlay with an image for a visually appealing hero section.
  • flexbox - Aligns sections horizontally and vertically for responsive design.
  • hover effects - Enhances interactivity by applying color changes and scaling images on hover.
  • media queries - Ensures the page layout adapts to different screen sizes.
  • box-shadow - Adds depth and distinction to sections, improving readability.

Accessibility

Accessibility considerations include:

  • Proper semantic HTML elements to improve screen reader compatibility.
  • Alternative text (alt) for all images, ensuring information is accessible to visually impaired users.
  • Contrast between text and background for readability.
  • Keyboard-navigable buttons and links for users relying on keyboard interactions.

Compliance

The web pages created adhere to the JANET regulations governing web page publication, ensuring compliance with accessibility, security, and ethical content standards. The following key aspects were considered:

  • Accessibility Compliance: The pages follow WCAG 2.1 guidelines by using semantic HTML, ensuring sufficient color contrast, and providing alternative text for images to accommodate users with disabilities.
  • Content Integrity: All information provided is factually accurate and appropriately cited where necessary, avoiding misinformation or misleading content.
  • Security Considerations: Best practices such as avoiding inline JavaScript for security vulnerabilities and ensuring proper use of external links (with target="_blank" and rel="noopener noreferrer") have been followed.
  • Privacy & Data Protection: No personal user data is collected or stored, ensuring compliance with GDPR principles.
  • Copyright & Attribution: Images and content sourced externally are properly attributed, following copyright regulations.

These measures ensure that the website aligns with ethical and legal web publishing standards while maintaining accessibility and usability for all users.
