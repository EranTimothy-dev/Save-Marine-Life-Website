Feedback Page

Technical Description

  • HTML:
    • Used <form> for structured user input.
    • Applied <fieldset> and <legend> to group related form controls for improved structure.
    • Integrated <button type="submit"> and <button type="reset"> for user actions.
    • Used <select> and <option> to provide dropdown selections.
    • Applied required attribute to essential fields: text input, email, and tel.
    • Ensured the form does not submit unless all required fields are completed.
    • Utilized <input type="text">, <input type="email">, <input type="tel">, <input type="radio">, <input type="checkbox">, <input type="file">, <input type="datetime-local">.
    • Ensured an intuitive and accessible feedback submission experience.
    • Allowed users to upload attachments and specify submission times.
    • Organized inputs using proper HTML structures to enhance usability.
    • Provided real-time feedback to users on incorrect inputs.

  • CSS:
    • Styled form elements using display: flex, margin, padding, and gap for spacing.
    • Used :hover pseudo-classes for better user experience.
    • Applied border: 2px solid red for invalid fields and displayed error messages.
    • Ensured form responsiveness using @media queries.
    • Applied border, border-radius, and box-shadow for better visual appeal.

Accessibility

  • Semantic HTML: Proper use of <header>, <section>, and <footer> for better screen reader navigation.
  • Alternative Text: Profile images include meaningful alt attributes to describe team members.
  • Keyboard Navigation: Ensure all interactive elements (like team member links) are accessible via Tab key.
  • Color Contrast: Background and text contrast meets WCAG standards for readability.
  • ARIA Labels: Used for team roles to help screen readers announce job titles effectively.

Compliance

The web pages created adhere to the JANET regulations governing web page publication, ensuring compliance with accessibility, security, and ethical content standards. The following key aspects were considered:

  • Accessibility Compliance: The pages follow WCAG 2.1 guidelines by using semantic HTML, ensuring sufficient color contrast, and providing alternative text for images to accommodate users with disabilities.
  • Content Integrity: All information provided is factually accurate and appropriately cited where necessary, avoiding misinformation or misleading content.
  • Security Considerations: Best practices such as avoiding inline JavaScript for security vulnerabilities and ensuring proper use of external links (with target="_blank" and rel="noopener noreferrer") have been followed.
  • Privacy & Data Protection: No personal user data is collected or stored, ensuring compliance with GDPR principles.
  • Copyright & Attribution: Images and content sourced externally are properly attributed, following copyright regulations.

These measures ensure that the website aligns with ethical and legal web publishing standards while maintaining accessibility and usability for all users.
