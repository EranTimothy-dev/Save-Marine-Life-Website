Splash Page

Technical Description

The splash page is designed with a full-screen video background, achieved using the <video> tag inside a <div> with position: fixed to ensure it covers the entire screen. The splash content is centrally aligned using display: flex, justify-content: center, and align-items: center.

Key HTML Elements

  • <video>: Plays a background video with autoplay and muted attributes for smooth display.
  • <div class="splash-screen">: The container holding the splash content.
  • <h1> and <p>: Used for branding and mission statements.

Key CSS Properties

  • object-fit: cover: Ensures the video fills the background properly.
  • filter: brightness(70%): Applies a dark overlay for better readability.
  • animation: waveBackground 6s infinite alternate ease-in-out: Creates a subtle scaling effect.
  • .loading-icon: A CSS-animated spinner to indicate loading.

Accessibility

The splash screen was designed with accessibility in mind to ensure usability for all users, including those with disabilities.

Color Contrast & Readability

The text is displayed in white against a dark background to provide high contrast, improving readability for users with visual impairments. A semi-transparent overlay (filter: brightness(70%)) further enhances the contrast between the text and background video.

Alternative Text for Images

The logo image includes an alt="Marine Logo" attribute, ensuring that screen readers can describe the image to visually impaired users.

Keyboard Navigation & Focus Management

The splash screen does not contain interactive elements, preventing unnecessary focus trapping. Future enhancements may include a "Skip to Main Content" link for improved navigation.

Motion Reduction Considerations

Animations such as the wave effect and loading spinner are smooth and slow, reducing motion sickness risks. The design respects the prefers-reduced-motion setting, allowing users to disable animations if needed.

Screen Reader Compatibility

The page structure uses semantic HTML elements like <h1>, <p>, and <div> to improve screen reader interpretation.

Time-Based Navigation Considerations

The splash screen redirects after 6 seconds using <meta http-equiv="refresh" content="6;url=home.html">, which may be problematic for users needing more time. Future improvements could include an option to skip the splash screen manually.

Volunteer Page

Technical Description

The Volunteer page is structured using semantic HTML elements and utilizes CSS Grid and Flexbox for responsive design. The page includes a hero section, event listings, and a review section.

Key HTML Elements

  • <div class="hero">: Contains a background image with a semi-transparent overlay for improved readability.
  • <div class="event-grid">: Uses CSS Grid to structure volunteer events into a visually appealing layout.
  • <section class="reviews">: Displays user reviews with a rating system.

Key CSS Properties

  • background: url(...): Sets a full-width background image in the hero section.
  • grid-template-columns: repeat(3, 1fr): Organizes the event cards into a responsive three-column layout.
  • .event-card:hover { transform: scale(1.05); }: Adds a hover effect for an interactive experience.
  • .rating-summary: Uses box-shadow and border-radius to create a clean review section layout.

Accessibility

The Volunteer page was designed with accessibility considerations to ensure usability for all users.

Color Contrast & Readability

The hero section uses a dark overlay on the background image, ensuring that text remains readable against various backgrounds.

Alternative Text for Images

The logo and event images include descriptive alt attributes to help screen reader users understand the content.

Keyboard Navigation & Focus Management

All interactive elements, such as buttons and links, are keyboard-navigable with visible focus indicators. The rating system and event sign-up buttons are accessible via the keyboard.

Motion Reduction Considerations

Hover animations and transitions are smooth and respect the prefers-reduced-motion media query, allowing users to disable animations if needed.

Content Page

Technical Description

The Content Page presents an informational blog about sustainable fishing, designed for readability and responsiveness. It is styled using a global CSS file (`styles.css`) along with a dedicated stylesheet (`ContentStyles.css`).

Key HTML Elements

  • <article class="blog-content">: Wraps the main blog content for better semantic structure.
  • <div class="blog-container">: Holds the blog content and sidebar elements.
  • <aside class="table-of-contents">: Provides a navigational table of contents for quick access to sections.
  • <button id="toc-toggle">: Allows toggling of the table of contents.

Key CSS Properties

  • display: flex: Used in the layout to position content and sidebar sections responsively.
  • background: url(...): Adds hero images to blog sections for better visual impact.
  • .blog-title: Uses larger fonts and margins to enhance readability.
  • .table-of-contents: Uses positioning and smooth scrolling to improve navigation.

Accessibility

The Content Page follows accessibility best practices to ensure a user-friendly experience for all.

Color Contrast & Readability

Text is displayed with high contrast over background images, ensuring easy readability.

Alternative Text for Images

All images (author profile, blog images, and icons) contain meaningful `` attributes to assist screen reader users.

Keyboard Navigation & Focus Management

The table of contents links, navigation bar, and footer links are fully keyboard-accessible.

Responsive & Scalable Design

The blog layout uses flexible grid structures, ensuring it adapts well on different screen sizes.

Challenges and Lessons Learned

During the implementation process, several challenges were encountered and addressed. Below are the key difficulties and how they were resolved:

1. Responsive Design Issues

Ensuring the layout adapted correctly across different screen sizes was a challenge. Solution: Used CSS Flexbox and Grid, and tested the site using media queries to optimize responsiveness.

2. Video Background Performance

The splash page’s video background caused slow load times on lower-end devices. Solution: Compressed the video, used a fallback image, and applied lazy loading techniques.

3. Cross-Browser Compatibility

Some animations and layout styles appeared differently in different browsers. Solution: Used CSS prefixes and tested with Chrome, Firefox, Edge, and Safari.

Lessons Learned

- The importance of structured debugging and cross-browser testing. - How to optimize performance while maintaining a visually appealing design. - Accessibility considerations should be prioritized from the start.

Compliance

The pages adhere to Janet regulations governing web page publication, ensuring compliance in the following areas:

1. Accessibility Standards

The site follows Janet regulations, including alt text for images, keyboard navigation, and sufficient color contrast.

2. Data Protection & Privacy

3. Copyright & Content Usage

4. Performance Optimization

The site uses compressed assets, lazy loading, and caching to ensure fast loading times.
